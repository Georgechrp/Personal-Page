Pokémon Explorer App

Εφαρμογή σε Kotlin με PokéAPI που επιτρέπει επιλογή τύπου Pokémon (Fire, Water, Grass, Electric, Dragon, Psychic, Ghost, Dark, Steel, Fairy), αναζήτηση ανά όνομα μέσα στον επιλεγμένο τύπο, προβολή λίστας με σταδιακό φόρτωμα (10-10), και λεπτομέρειες (όνομα, εικόνα, βασικά stats: HP, Attack, Defense).

Σύνοψη

Περιγραφή:

Να υλοποιηθεί mobile app (Kotlin/Android) που καταναλώνει το PokéAPI για 10 προκαθορισμένους τύπους, υποστηρίζει αναζήτηση εντός του τύπου, προβάλει 10-10 αποτελέσματα και προσφέρει βασικές λεπτομέρειες Pokémon.

Κύρια χαρακτηριστικά:
  • Επιλογή τύπου από Fire, Water, Grass, Electric, Dragon, Psychic, Ghost, Dark, Steel, Fairy
  • Αναζήτηση ανά όνομα μέσα στον επιλεγμένο τύπο
  • Λίστα με "Load more" (10-10) από PokéAPI
  • Λεπτομέρειες Pokémon: εικόνα (official artwork), HP, Attack, Defense
  • Κατάσταση φόρτωσης, σφάλματα, κενά αποτελέσματα, retry

Προσέγγιση & Αρχιτεκτονική

Stack & Βιβλιοθήκες
  • Kotlin, AndroidX, Material 3
  • Coroutines + Flow για async
  • Retrofit + OkHttp + Moshi
  • Coil για εικόνες
  • Hilt για DI
  • Jetpack Compose για UI ή XML + RecyclerView (εναλλακτικά)
Δομή (Clean Architecture)
  • Data: API services, DTOs, mappers, caches
  • Domain: use cases, entities
  • Presentation: ViewModels (Flow), UI screens
  • Sealed state: Loading/Content/Error/Empty
API & Pagination
  • Λήψη Pokémon per type: /type/{type}
  • Βαθμιαία λήψη details ανά 10 (HP/ATK/DEF + image)
  • Local cache για αποφυγή διπλών calls
  • Search εφαρμόζεται στο filtered set του τύπου (client-side)

Ποιότητα, Testing & Εμπειρία Χρήστη

UX/States
  • Skeletons/Spinners σε αρχικό φόρτωμα και “Load more”
  • Empty states με καθοδήγηση
  • Error states με retry ανά block
  • Θυμάται τον επιλεγμένο τύπο και το scroll position
Performance
  • Coil image caching + placeholders
  • Batching detail requests (10-10)
  • Debounced search (300ms)
  • Memoization & in-memory cache per type
Testing
  • Unit tests για mappers/use cases
  • ViewModel tests με Turbine
  • UI tests (Compose testing ή Espresso)
  • Contract tests για API models

Εκτίμηση χρόνου υλοποίησης

1) Θεμελίωση (0.5–1 μέρα)
  • Project setup, Hilt, Retrofit, modules
  • Models/DTOs, mappers, API contracts
2) Λογική & UI (1–1.5 μέρες)
  • Type selection + search με debounce
  • Pagination 10-10 και caching
  • Details screen με HP/ATK/DEF & εικόνες
3) Polish & Tests (0.5–1 μέρα)
  • Empty/error/loading states
  • Unit/UI tests, accessibility
  • Readme + μικρές βελτιώσεις

Σύνολο: περίπου 2–3 εργάσιμες ημέρες (16–24 ώρες), ανάλογα με το επίπεδο polish και τις τυχόν διευκρινίσεις.

Απορίες / Διευκρινίσεις

  • Για την αναζήτηση: αποδεκτή είναι case-insensitive contains; Να υποστηριχθεί και αρχικογράμματα/accents handling;
  • Απαιτήσεις για offline λειτουργία; Εάν ναι, προτείνεται Room cache per type.
  • Σχεδιαστική κατεύθυνση: Material 3 light/dark ή custom theme;
  • Στόχος Android minSdk/targetSdk;
  • Περιορισμοί rate limiting του PokéAPI: Να προστεθεί exponential backoff;
